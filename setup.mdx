This guide covers how to set up Sift on your local machine for development. You'll install dependencies, configure environment variables, populate the database with real course data, and run the dev servers.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 20+** installed
- A **Supabase** project (free tier is fine)
- An **Anthropic API key** for transcript parsing (get one at [console.anthropic.com](https://console.anthropic.com))
- A **Clerk** account for authentication (get one at [clerk.com](https://clerk.com))
- Git installed

## Step 1: Clone the Repository

```bash
git clone https://github.com/your-org/sift.git
cd sift
```

## Step 2: Install Dependencies

Sift uses a monorepo structure with separate client and server packages.

```bash
# Install root dependencies (concurrently for running both servers)
npm install

# Install client dependencies
cd client && npm install

# Install server dependencies
cd ../server && npm install
```

## Step 3: Configure Environment Variables

You'll need to create environment files for both the client and server.

### Client Environment

Create `client/.env.local`:

```bash
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=<your-clerk-publishable-key>
CLERK_SECRET_KEY=<your-clerk-secret-key>
```

<Note>
Get your Clerk keys from the [Clerk Dashboard](https://dashboard.clerk.com). Create a new application if you haven't already.
</Note>

### Server Environment

Create `server/.env`:

```bash
PORT=3001
FRONTEND_URL=http://localhost:3000
SUPABASE_URL=<your-supabase-url>
SUPABASE_ANON_KEY=<your-supabase-anon-key>
ANTHROPIC_API_KEY=<your-anthropic-api-key>
CLERK_PUBLISHABLE_KEY=<your-clerk-publishable-key>
CLERK_SECRET_KEY=<your-clerk-secret-key>
```

<Warning>
**Never commit `.env` or `.env.local` files to version control.** These files contain sensitive credentials.
</Warning>

## Step 4: Set Up the Database

### Create Tables

Sift uses the following database schema:

- `courses` — course metadata (department, number, name, credits, prerequisites)
- `professors` — professor data (name, RMP ratings, difficulty, tags)
- `grade_distributions` — historical grade data by course, professor, and semester
- `sections` — current semester sections (professor, time slot, enrollment)
- `degree_plans` — degree requirements by major and catalog year
- `user_profiles` — user preferences, completed courses, in-progress courses

You'll need to create these tables in your Supabase project. See the `supabase/` directory in the repository for migration files.

<Note>
If migration files aren't included, you can run the scraper (Step 5) which will attempt to insert into these tables. Supabase will auto-create tables if you have the right permissions, but it's better to define the schema explicitly.
</Note>

## Step 5: Populate the Database with Course Data

Sift includes a scraper that pulls real data from Anex and RateMyProfessors.

### Run the Scraper

```bash
cd server
npm run scrape
```

This script:

1. **Scrapes Anex** — fetches grade distributions for all CSCE courses (110, 111, 120, 121, 221, 222, 310, etc.)
2. **Scrapes RateMyProfessors** — fetches ratings for all CS department faculty at Texas A&M
3. **Aggregates data** — groups grade distributions by course, professor, and semester
4. **Inserts into Supabase** — populates `courses`, `professors`, `grade_distributions`, and `degree_plans` tables

```typescript Anex Scraper
async function fetchAnex(dept: string, num: string): Promise<AnexRecord[]> {
  const body = new URLSearchParams({ dept, number: num }).toString();
  const url = new URL(ANEX_URL);

  return new Promise((resolve) => {
    const req = https.request(
      {
        hostname: url.hostname,
        path: url.pathname,
        method: "POST",
        headers: {
          "Content-Type": "application/x-www-form-urlencoded",
          "Content-Length": Buffer.byteLength(body),
        },
        agent: anexAgent, // TLS bypass scoped to Anex only
      },
      (res) => {
        let data = "";
        res.on("data", (chunk) => { data += chunk; });
        res.on("end", () => {
          try {
            const parsed = JSON.parse(data);
            resolve(parsed.classes || []);
          } catch {
            resolve([]);
          }
        });
      }
    );

    req.on("error", () => resolve([]));
    req.write(body);
    req.end();
  });
}
```

```typescript RateMyProfessors Scraper
async function fetchAllRMPTeachers(): Promise<RMPTeacher[]> {
  const teachers: RMPTeacher[] = [];
  let cursor: string | null = null;
  let hasNext = true;
  let page = 0;

  while (hasNext) {
    page++;
    const afterClause: string = cursor ? `after: "${cursor}"` : "";

    const query: string = `
      query {
        newSearch {
          teachers(
            query: { text: "", schoolID: "${TAMU_SCHOOL_ID}", departmentID: "${CS_DEPT_ID}" }
            first: 20
            ${afterClause}
          ) {
            resultCount
            pageInfo { hasNextPage endCursor }
            edges {
              node {
                id firstName lastName department
                avgRatingRounded avgDifficultyRounded numRatings
                wouldTakeAgainPercentRounded legacyId
                teacherRatingTags { tagName tagCount }
              }
            }
          }
        }
      }
    `;

    const res: Response = await fetch(RMP_URL, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Authorization: RMP_AUTH,
        "User-Agent": "Mozilla/5.0",
      },
      body: JSON.stringify({ query }),
    });

    const data: any = await res.json();
    const connection: any = data?.data?.newSearch?.teachers;

    if (!connection) {
      console.error("  RMP query failed:", JSON.stringify(data?.errors));
      break;
    }

    for (const edge of connection.edges) {
      teachers.push(edge.node);
    }

    hasNext = connection.pageInfo.hasNextPage;
    cursor = connection.pageInfo.endCursor;

    console.log(
      `  Page ${page}: +${connection.edges.length} teachers (total: ${teachers.length})`
    );
    await sleep(200);
  }

  return teachers;
}
```

## Common Issues

### "Missing SUPABASE_URL" error

Ensure `server/.env` exists and contains valid Supabase credentials:

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
```

From server/src/db/supabase.ts:3-6

### Transcript parsing fails

Check that:

1. `ANTHROPIC_API_KEY` is set in `server/.env`
2. Your API key is valid and has sufficient credits
3. The uploaded PDF is a valid TAMU transcript (starts with `%PDF` magic bytes)

From server/src/routes/transcript.ts:49-53

### Recommendations are empty

Ensure:

1. You've run the scraper to populate the database
2. You've uploaded a transcript with completed courses
3. There are courses available for the selected semester
