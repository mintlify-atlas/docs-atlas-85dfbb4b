## Get Degree Plan

<div className="api-method">
  <div className="api-method-badge get">GET</div>
  <code>/api/degree-plan/:major</code>
</div>

Retrieve the complete degree plan for a specific major, including all requirements, course lists, and credit distributions.

### Path Parameters

<ParamField path="major" type="string" required>
  Major code (e.g., "CS", "CPEN", "MATH"). Must be alphabetic, maximum 10 characters.
</ParamField>

### Query Parameters

<ParamField query="catalog_year" type="string" optional>
  Specific catalog year (e.g., "2024-2025"). If not provided, returns the most recent catalog year.
</ParamField>

### Response

Returns the degree plan for the specified major.

<ResponseField name="data" type="DegreePlan">
  Degree plan object
  
  <Expandable title="DegreePlan properties">
    <ResponseField name="id" type="string">
      Degree plan UUID
    </ResponseField>
    
    <ResponseField name="major" type="string">
      Major code
    </ResponseField>
    
    <ResponseField name="catalog_year" type="string">
      Catalog year (e.g., "2024-2025")
    </ResponseField>
    
    <ResponseField name="requirements" type="DegreeRequirement[]">
      Array of degree requirements
      
      <Expandable title="DegreeRequirement properties">
        <ResponseField name="type" type="string">
          Requirement type: "core", "elective", "lab", "math", "science", or "capstone"
        </ResponseField>
        
        <ResponseField name="category" type="string">
          Human-readable category name (e.g., "Core Computer Science")
        </ResponseField>
        
        <ResponseField name="courses" type="string[]">
          List of course codes in this requirement (e.g., ["CSCE 121", "CSCE 221"])
        </ResponseField>
        
        <ResponseField name="credits_needed" type="number">
          Total credits required for this category
        </ResponseField>
        
        <ResponseField name="selection_rule" type="string" optional>
          "all" (must complete all courses) or "pick" (choose courses totaling credits_needed). Defaults to "all".
        </ResponseField>
        
        <ResponseField name="equivalents" type="object" optional>
          Mapping of course codes to equivalent alternatives. Key is the canonical course, value is array of equivalents.
        </ResponseField>
        
        <ResponseField name="track" type="string" optional>
          Track or concentration name if this requirement is track-specific
        </ResponseField>
        
        <ResponseField name="credits_map" type="object" optional>
          Custom credit values for specific courses (overrides default of 3). Key is course code, value is credits.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="error" type="string | null">
  Error message if request failed, otherwise null
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.sift.app/api/degree-plan/CS?catalog_year=2024-2025"
```

```javascript JavaScript
const response = await fetch(
  'https://api.sift.app/api/degree-plan/CS?catalog_year=2024-2025'
);
const { data, error } = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.sift.app/api/degree-plan/CS',
    params={'catalog_year': '2024-2025'}
)
result = response.json()
```
</CodeGroup>

### Example Response

```json
{
  "data": {
    "id": "plan-123",
    "major": "CS",
    "catalog_year": "2024-2025",
    "requirements": [
      {
        "type": "core",
        "category": "Core Computer Science",
        "courses": ["CSCE 121", "CSCE 221", "CSCE 310"],
        "credits_needed": 11,
        "selection_rule": "all",
        "credits_map": {
          "CSCE 121": 4,
          "CSCE 221": 4,
          "CSCE 310": 3
        }
      },
      {
        "type": "elective",
        "category": "Technical Electives",
        "courses": ["CSCE 411", "CSCE 420", "CSCE 431", "CSCE 441"],
        "credits_needed": 9,
        "selection_rule": "pick"
      }
    ]
  },
  "error": null
}
```

<Note>
  If no `catalog_year` is specified, the API returns the most recent degree plan for the major.
</Note>

<Warning>
  Returns 404 if no degree plan exists for the specified major (or catalog year, if provided).
</Warning>

## Calculate Remaining Requirements

<div className="api-method">
  <div className="api-method-badge post">POST</div>
  <code>/api/degree-plan/remaining</code>
</div>

Compute your remaining degree requirements based on completed and in-progress courses. This endpoint intelligently handles course equivalents and "pick N credits" requirements.

### Request Body

<ParamField body="major" type="string" required>
  Major code (e.g., "CS"). Maximum 10 characters.
</ParamField>

<ParamField body="completed_courses" type="string[]" required>
  Array of completed course codes (e.g., ["CSCE 121", "MATH 151"]). Maximum 200 courses, each up to 20 characters.
</ParamField>

<ParamField body="in_progress_courses" type="string[]" default="[]">
  Array of courses currently in progress. Maximum 50 courses.
</ParamField>

### Response

Returns a detailed breakdown of remaining requirements with progress tracking.

<ResponseField name="data" type="object">
  Remaining requirements analysis
  
  <Expandable title="Properties">
    <ResponseField name="plan" type="DegreePlan">
      Full degree plan object (see above)
    </ResponseField>
    
    <ResponseField name="remaining" type="object[]">
      Array of requirement categories with progress details
      
      <Expandable title="Category properties">
        <ResponseField name="type" type="string">
          Requirement type
        </ResponseField>
        
        <ResponseField name="category" type="string">
          Category name
        </ResponseField>
        
        <ResponseField name="courses" type="string[]">
          All courses in this category (from degree plan)
        </ResponseField>
        
        <ResponseField name="credits_needed" type="number">
          Total credits required
        </ResponseField>
        
        <ResponseField name="remaining_courses" type="string[]">
          Courses still needed
        </ResponseField>
        
        <ResponseField name="completed_courses" type="string[]">
          Courses you've completed in this category
        </ResponseField>
        
        <ResponseField name="in_progress_courses" type="string[]">
          Courses you're currently taking in this category
        </ResponseField>
        
        <ResponseField name="credits_completed" type="number">
          Credits earned in this category
        </ResponseField>
        
        <ResponseField name="credits_in_progress" type="number">
          Credits currently in progress
        </ResponseField>
        
        <ResponseField name="is_satisfied" type="boolean">
          Whether this requirement is fully satisfied (completed courses only)
        </ResponseField>
        
        <ResponseField name="equivalent_matches" type="object" optional>
          Mapping showing which canonical courses were satisfied by equivalents (key: canonical course, value: equivalent taken)
        </ResponseField>
        
        <ResponseField name="selection_rule" type="string" optional>
          "all" or "pick"
        </ResponseField>
        
        <ResponseField name="equivalents" type="object" optional>
          Equivalency map from degree plan
        </ResponseField>
        
        <ResponseField name="track" type="string" optional>
          Track name if applicable
        </ResponseField>
        
        <ResponseField name="credits_map" type="object" optional>
          Custom credit values
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="total_credits_required" type="number">
      Total credits needed to graduate
    </ResponseField>
    
    <ResponseField name="total_credits_completed" type="number">
      Total credits completed (capped per category at credits_needed to avoid over-counting)
    </ResponseField>
    
    <ResponseField name="total_credits_in_progress" type="number">
      Total credits currently in progress
    </ResponseField>
    
    <ResponseField name="progress_pct" type="number">
      Graduation progress percentage (0-100)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="error" type="string | null">
  Error message if request failed, otherwise null
</ResponseField>

### Example Request

<CodeGroup>
```bash cURL
curl -X POST "https://api.sift.app/api/degree-plan/remaining" \
  -H "Content-Type: application/json" \
  -d '{
    "major": "CS",
    "completed_courses": ["CSCE 121", "MATH 151", "PHYS 206"],
    "in_progress_courses": ["CSCE 221", "MATH 251"]
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sift.app/api/degree-plan/remaining', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    major: 'CS',
    completed_courses: ['CSCE 121', 'MATH 151', 'PHYS 206'],
    in_progress_courses: ['CSCE 221', 'MATH 251']
  })
});
const { data, error } = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.sift.app/api/degree-plan/remaining',
    json={
        'major': 'CS',
        'completed_courses': ['CSCE 121', 'MATH 151', 'PHYS 206'],
        'in_progress_courses': ['CSCE 221', 'MATH 251']
    }
)
result = response.json()
```
</CodeGroup>

<Note>
  **Equivalency handling:** If you take an equivalent course (e.g., ENGR 102 instead of CSCE 111), the algorithm automatically recognizes it and marks the canonical course as satisfied. The `equivalent_matches` field shows which substitutions were made.
</Note>

<Note>
  **"Pick" requirements:** For categories with `selection_rule: "pick"` (e.g., "choose 9 credits from these electives"), the category is marked as satisfied once you've completed enough credits, and remaining courses become empty.
</Note>

<Warning>
  Only completed courses count toward `is_satisfied` and `progress_pct`. In-progress courses are tracked separately to show projected progress.
</Warning>
